# LangGraph 기반 운동 추천 멀티 에이전트 시스템

사용자의 신체 상태를 분석해 맞춤 운동을 추천하는 멀티 에이전트 파이프라인입니다.  
LangGraph의 상태 기반 그래프 구조를 활용해 에이전트 간 데이터 흐름을 명시적으로 설계했습니다.

## 프로젝트 개요

자연어로 입력된 신체 상태 질문을 3단계 에이전트가 순차적으로 처리합니다.

- 단순 LLM 호출이 아닌, 역할이 분리된 에이전트 구조로 응답 품질과 추적 가능성을 높였습니다.
- LangGraph의 `StateGraph`를 사용해 에이전트 간 상태(state)를 명시적으로 전달합니다.
- 로컬 LLM(Ollama)을 연동해 외부 API 없이 완전히 로컬에서 동작합니다.

## 기술 스택

| 분류 | 사용 기술 |
|------|----------|
| 에이전트 오케스트레이션 | LangGraph |
| LLM 프레임워크 | LangChain |
| 로컬 LLM 서버 | Ollama (exaone3.5:2.4b) |
| 언어 | Python 3.10 |
| 실행 환경 | Jupyter Notebook / VS Code |

## 멀티 에이전트 구조

3개의 에이전트가 LangGraph의 방향 그래프(DAG) 위에서 순서대로 실행됩니다.


[사용자 질문]
      |
      v
 extractor   →   신체 상태 키워드 추출
      |
      v
  matcher    →   증상 해결에 효과적인 운동 5가지 후보 선정
      |
      v
  answer     →   개조식 운동 추천 답변 생성
      |
      v
 [최종 출력]


### 에이전트별 역할

**extractor (추출 에이전트)**
- 사용자 질문에서 신체 상태/증상 키워드만 추출
- 예) "체력이 안 좋고 살이 계속 찐다" → `체력, 살 찌는`

**matcher (후보 에이전트)**
- 추출된 증상을 기반으로 효과적인 운동 5가지를 선정
- 예) `체력, 살 찌는` → `유산소 운동, 저항성 훈련, HIIT ...`

**answer (답변 에이전트)**
- 증상과 운동 목록을 받아 개조식 자연어 답변 생성
- 마크다운/이모지 없이 사람이 쓴 것처럼 출력되도록 프롬프트 설계

## 실행 결과

> 입력: `"체력이 안 좋고 살이 계속 찌는데 어떤 운동을 할까?"`

```
============================== 추출된 증상:
체력, 살 찌는

============================== 추천 운동 목록:
심폐지구력 향상 운동, 저항성 훈련, 유산소 운동, 균형 잡힌 식단 조절, 고강도 인터벌 트레이닝

============================== 최종 답변:
심폐지구력 향상 운동
- 버피 테스트: 전신을 활용한 복합 동작으로 반복 수행
- 심폐 기능 강화 및 체력 전반 향상에 효과적

저항성 훈련
- 덤벨이나 바벨을 활용한 근력 운동
- 근육량 증가로 기초대사량 높여 체중 관리에 유리

유산소 운동
- 빠르게 걷기 또는 가벼운 달리기, 하루 30분 이상
- 체지방 소모와 심혈관 건강 개선에 효과

HIIT (고강도 인터벌 트레이닝)
- 고강도 운동과 휴식을 짧게 반복
- 짧은 시간 안에 높은 칼로리 소모 가능
```

## 실행 방법

**1. 패키지 설치**
```bash
pip install grandalf langchain langgraph langchain-community
```

**2. Ollama 서버 실행 및 모델 다운로드**
```bash
ollama serve &
ollama pull exaone3.5:2.4b
```

**3. 노트북 실행**
```bash
jupyter notebook langgraph_exercise.ipynb
```

## 핵심 설계 포인트

**상태 기반 파이프라인**  
`TypedDict`로 정의한 `AgentState`가 각 에이전트를 거치며 점진적으로 채워집니다.  
어느 단계에서 무엇이 생성됐는지 추적이 쉽고, 에이전트 교체나 추가가 용이합니다.

**프롬프트 엔지니어링**  
답변 에이전트에 출력 형식 규칙(마크다운 금지, 구어체, 개조식)을 명시해  
LLM 특유의 AI 말투 없이 자연스러운 결과를 유도했습니다.

**로컬 LLM 연동**  
Ollama를 통해 외부 API 없이 로컬에서 완전 동작합니다.  
모델명 한 줄만 바꾸면 다른 Ollama 모델로 즉시 교체 가능합니다.
